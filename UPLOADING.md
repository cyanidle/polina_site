# Как загружать контент

## Комикс

```
comics/ru/Полный выплатец/
├── meta.json          ← необязательно
├── teaser/            ← необязательно, картинки для карусели на странице комикса
│   ├── 1.png
│   └── 2.png
├── 0. Обложка.png
├── 1стр.png
├── 2стр.png
└── ...
```

`meta.json`:

```json
{
  "title": "Полный выплатец",
  "description": "О чём комикс.",
  "cover": "0. Обложка.png",
  "characters": [
    { "name": "Guccio Belatti", "about": "Кто это" }
  ],
  "pages": {
    "3стр.png": { "comment": "Комментарий автора", "date": "2026-01-15" }
  }
}
```

Все поля необязательны. Без `meta.json` название = имя папки, обложка = первая страница по алфавиту/номеру.

Для английской версии — та же папка, но под `comics/en/`.

### Картинки для карусели (`teaser/`)

Карусель на странице комикса (обложка + несколько кадров) берёт картинки из подпапки `teaser/` — это отдельные картинки (например, без текста), а не страницы комикса. Они показываются в том порядке, в котором отсортированы их имена файлов (см. «Формат страниц» ниже). Без этой папки в карусели будет только обложка.

## Перевод текста в одном meta.json

Если картинки одни и те же, а разница только в тексте — `title`, `description`, `about` персонажа и `comment` страницы можно вместо строки задать объектом с языками. Сайт сам подставит нужный язык в зависимости от переключателя RU/EN:

```json
{
  "title": { "ru": "Полный выплатец", "en": "Paid in Full" },
  "description": { "ru": "О чём комикс.", "en": "What it's about." },
  "pages": {
    "3стр.png": {
      "comment": { "ru": "Комментарий автора", "en": "Author's comment" },
      "date": "2026-01-15"
    }
  }
}
```

## Комикс с главами

Если внутри папки комикса лежат подпапки — каждая подпапка становится отдельной главой:

```
comics/ru/Мой комикс/
├── meta.json
├── Глава 1/
│   ├── 1.png
│   └── 2.png
└── Глава 2/
    ├── 1.png
    └── 2.png
```

(`teaser/` в этот список не попадает — это зарезервированное имя для картинок карусели, а не глава.)

## Арт

```
arts/Закат.png
arts/Закат.txt   ← необязательное описание
```

Название = имя файла. Файл `.txt` с тем же именем — текст описания, который покажется рядом с работой.

## Персонажи

Отдельная вкладка «Персонажи» на главной. Каждый персонаж — это **отдельная папка** внутри `characters/`, а в ней сколько угодно картинок. К каждой картинке можно положить `.txt` с тем же именем — это подпись, которая покажется под ней:

```
characters/Guccio Belatti/
├── 1 портрет.png
├── 1 портрет.txt      ← необязательная подпись к этой картинке
├── 2 в бою.png
├── 2 в бою.txt        ← необязательная подпись
└── 3 арт.png          ← без подписи — тоже можно
```

Имя персонажа = имя папки. Обложка в списке — первая картинка (по порядку имён, см. «Формат страниц»). Пустые папки пропускаются.

Если в `meta.json` комикса у персонажа `name` **совпадает** с именем папки (регистр не важен), то на странице комикса его имя станет ссылкой на страницу этого персонажа. Например, персонаж `{ "name": "Guccio Belatti" }` свяжется с папкой `characters/Guccio Belatti/`.

## Формат страниц

Имена файлов сортируются "по-виндовски" (`2.png` раньше `10.png`), нумерация с ведущими нулями не нужна.

## Большие картинки

Крупные изображения (страницы, арты, картинки персонажей) сервер уменьшает автоматически — отдельно ничего делать не нужно. Рядом с оригиналом появляется уменьшенная копия с суффиксом `-sm` (например, `1стр.png` → `1стр-sm.png`), и в выдаётся именно она. Оригинал не меняется.

- Порог размера, суффикс и остальные параметры настраиваются на сервере (см. `IMAGE_RESIZE_*` в README).
- Эти `-sm` файлы создавать вручную не надо — и не называйте так свои файлы: всё, оканчивающееся на `-sm`, сервер считает уменьшенной копией и не показывает в списке.
