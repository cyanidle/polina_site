# Как загружать контент

Весь контент лежит в трёх папках:

```text
comics/       комиксы
arts/         отдельные арты
characters/   галереи персонажей
```

Сервер замечает новые и заменённые файлы автоматически. Перезапуск обычно не нужен.

## Комикс без глав

Создайте папку внутри `comics/ru/` или `comics/en/`:

```text
comics/ru/Полный выплатец/
├── meta.json              необязательно
├── teaser/                необязательно, карусель на странице комикса
│   ├── 1.png
│   └── 2.png
├── 0. Обложка.png
├── 1стр.png
├── 2стр.png
└── 3стр.png
```

Все картинки в корне папки комикса считаются страницами, включая обложку. `teaser/` в читалку не попадает.

## Порядок страниц

Порядок всегда определяется именами файлов с естественной сортировкой:

```text
1стр.png
2стр.png
3стр.png
10стр.png
```

То есть `2стр.png` идёт раньше `10стр.png`; ведущие нули не обязательны. Используйте уникальный последовательный номер в начале каждого имени. Не рассчитывайте на дату создания файла или порядок загрузки в папку.

Например, имена `4стр.png`, `4стр1ч.png`, `4стр2ч.png`, `5стр.png` будут показаны именно в таком порядке. Если нужен другой порядок, переименуйте исходные файлы. Отдельного списка порядка нет.

Папки `teaser/` и `small/` зарезервированы и не считаются главами.

## Комикс с главами

Если в папке комикса есть другие подпапки, каждая непустая подпапка становится главой:

```text
comics/ru/Мой комикс/
├── meta.json
├── Глава 1/
│   ├── 1.png
│   └── 2.png
└── Глава 2/
    ├── 1.png
    └── 2.png
```

Главы и страницы внутри главы также сортируются естественно по имени.

Не смешивайте главы и страницы в корне. Как только найдена хотя бы одна папка главы, картинки в корне комикса игнорируются. Сервер пишет об этом предупреждение в журнал.

## `meta.json`

Все поля необязательны:

```json
{
  "title": "Полный выплатец",
  "description": "О чём комикс.",
  "cover": "0. Обложка.png",
  "characters": [
    { "name": "Guccio Belatti", "about": "Кто это" }
  ],
  "pages": {
    "3стр.png": {
      "comment": "Комментарий автора",
      "date": "2026-01-15"
    }
  }
}
```

Без `meta.json` названием будет имя папки, обложкой — первая страница.

В комиксе с главами указывайте путь от папки комикса. Это особенно важно, если в разных главах повторяются имена `1.png`, `2.png`:

```json
{
  "cover": "Глава 1/1.png",
  "pages": {
    "Глава 1/1.png": { "comment": "Начало первой главы" },
    "Глава 2/1.png": { "comment": "Начало второй главы" }
  }
}
```

Короткие ключи вроде `"1.png"` поддерживаются для старых файлов, но при повторяющихся именах применятся ко всем совпадениям.

Если JSON повреждён, сервер игнорирует метаданные и пишет предупреждение в журнал. Проверяйте запятые, кавычки и закрывающие скобки.

## Переводы

`title`, `description`, `about` и `comment` могут быть строкой или объектом с переводами:

```json
{
  "title": { "ru": "Полный выплатец", "en": "Paid in Full" },
  "description": { "ru": "О чём комикс.", "en": "What it is about." },
  "pages": {
    "3стр.png": {
      "comment": { "ru": "Комментарий автора", "en": "Author's comment" },
      "date": "2026-01-15"
    }
  }
}
```

Переключатель RU/EN выбирает разные списки: `comics/ru/` и `comics/en/`. Переводы внутри одного `meta.json` меняют текст при открытии этого комикса, но сами по себе не добавляют его в список другого языка. Чтобы комикс появился в английском списке, нужна папка в `comics/en/`.

## Карусель

Положите дополнительные картинки в `teaser/`:

```text
comics/ru/Мой комикс/teaser/
├── 1.png
├── 2.png
└── 10.png
```

Они показываются после обложки в естественном порядке и не считаются страницами.

## Арты

```text
arts/Закат.png
arts/Закат.txt       необязательное описание
```

Название берётся из имени картинки. Текстовый файл должен иметь такое же имя без расширения картинки.

## Персонажи

Каждый персонаж — отдельная папка:

```text
characters/Guccio Belatti/
├── 1 портрет.png
├── 1 портрет.txt        необязательная подпись
├── 2 в бою.png
├── 2 в бою.txt
└── 3 арт.png
```

Имя персонажа берётся из имени папки. Первая картинка в естественном порядке становится обложкой. Пустые папки не показываются.

Чтобы связать персонажа с комиксом, его `name` в `meta.json` должен совпадать с именем папки без учёта регистра:

```json
{
  "characters": [
    { "name": "Guccio Belatti", "about": "Описание" }
  ]
}
```

## Как заменить картинку

Чтобы исправить рисунок и сохранить его место в комиксе:

1. Замените исходный файл, сохранив то же имя.
2. Не трогайте копию в `small/`: сервер обновит её сам.
3. Не сохраняйте старую дату изменения при копировании. Если программа это сделала, выполните `touch "путь/к/файлу.png"`.
4. Подождите несколько секунд и обновите страницу.

Адрес картинки содержит версию исходного файла, поэтому обычная замена сразу обходит старый кэш. На сервере дополнительно установлен короткий кэш до пяти минут с перепроверкой.

Если старая картинка всё же осталась:

```bash
touch "comics/ru/Мой комикс/3стр.png"
./scripts/purge-small-dirs.sh
```

После этого сервер пересоздаст `small/`. В браузере можно сделать принудительное обновление страницы.

## Папка `small/`

При включённом уменьшении сервер создаёт WebP-копии больших картинок:

```text
3стр.png
small/3стр.webp
```

`small/` — служебный кэш. Не загружайте туда оригиналы, не редактируйте эти файлы вручную и не используйте это имя для главы. Папку можно безопасно удалить: сервер создаст её заново.

Поддерживаемые форматы изображений: PNG, JPG/JPEG, GIF, WebP, BMP и SVG.
